3. ¿Qué son los request y response?
response: es un objeto para presentar en la pantalla
del navegador del cliente el resultado de cualquier
codigo que hayamos escrito
request: El objeto request permite el acceso a toda
informacion que pasa desde el navegador del cliente al servidor.
4. ¿Qué son los códigos HTTP (200, 400, 500) ?
200: es un codigo de respuesta del servidor http que nos ofrecera el status
ante una peticion estandar correcta a la que puede responder sin problemas.
400: son solicitudes incorrectas cuando un servidor no puede entender una
solicitud que se ha hecho de el.
500: es un codigo de estado http muy general que significa que algo
ha ido mal en el servidor del sitio web, pero el servidor no puede
ser mas especifico sobre el cual es el problema exacto.
